Περιεχόμενα:

Η εργασια αποτελειται απο 25 αρχεια.19 απο αυτα ειναι .c αρχεια 5 .h και 1 Makefile.Επελεξα να γραψω ξεχωριστα τις συναρτησεις ανα κατηγοριες (πχ ολες οι συναρτησεις για hash table σε ενα αρχειο) ωστε να ειναι πιο ευκολο να δειτε τις λειτουργιες τους απομονομενα αντι να ψαχνεται σε ενα μεγαλο αρχειο με ολες τις συναρτησεις.

Παρατηρησεις:

Το προγραμμα εχει ελεγχθει με valgrind και σε οσες περιπτωσεις εχω ελεγξει καταφερνει επιτυχως να αποδεσμευσει ολη τη μνημη που χρησιμοποιει.

Επισης ειναι ικανο να τρεξει ακομα και αν επιλεξεται στο configuration file να βαλεται μονο μια θεση για το  hash table απλα στα μεγαλα αρχεια παιρνεια παρα πολυ χρονο για να εκτελεστει και ειναι πρακτικα ασκοπο να δοκιμαστει αλλα λειτουργει.

Το config file για να ειναι αποδεκτο θα πρεπει να περιεχει εναν ακεραιο αριθμο μονο και τιποτα αλλο.Οτιδηποτε αλλο θα εχει ως αποτελεσμα το προγραμμα να χρησιμοποιησει μια default τιμη για τον πινακα αν δεν υπαρχει και input file. 

Το μεγεθος του πινακα καθοριζεται ετσι ωστε να ειναι αποδοτικο με βαση τα δεδομενα του.Στη περιπτωση που εχουμε config file προφανως το size του ειναι οτι διαβαστει απο το αρχειο.Ωστοσο αν δεν υπαρχει το προγραμμα διαβαζει μια φορα το Input file και αποθηκευει το συνολικο αριθμο εγγραφων σε μια μεταβλητη. Μετα καθοριζει το size του πινακα με εναν απλο τροπο.Απο τις δομες γνωριζουμε οτι για να ειναι αποδοτικος ενας hashtable με seperate chaining πρεπει το load factor να ειναι μικροτερο του 0.9. Χρησιμποιοωντας αυτη τη πληροφορια εχω βαλει το load factor να ειναι απο default 0.6 (θα εβαζα και μικροτερο αλλα μετα νομιζω οτι θα γινεται σπαταλη πορων) ετσι το size προκυπτει απο την εξης ισοτητα:

size = (αριθμος εγγραφων) / (load factor)

Τελος αν δεν υπαρχει το input file το μεγεθος του πινακα καθοριζεται αντικαθιστώντας τον αριθμο εγγραφων με μια default μεταβλητη.

Περα απο τις βασικες δομες που πρεπει να υλοποιησουμε εχω ακομα μια inverted index δομη η οποια αυτη τη φορα δημιουργειτα με βαση τα postal code για να με διευκοληνει με την postalcode συναρτηση.

Και στις δυο λιστες (year list και post list) υπαρχει και μια μεταβλητη counter που κανει διευκοληνη στις συναρτησεις count και postalcode.

Και οι δυο λιστες ειναι ταξινομημενες.Αυτο σημαινει για τη year list οτι ο πρωτος κομβος εχει το μεγαλυτερο ετος και ο τελευταιος το μικροτερο. Η postal code απο την αλλη ταξινομειται με βαση το counter της ωστε να μας βοηθησει να βρουμε ευκολα το rank.

Τελος η postal code ειναι διπλα συνδεδεμενη λιστα για να μας επιτρεπει να συγκρινουμε και να μετακινουμε τους κομβους καλυτερα.

Η εντολη εκτελεσης μπορει να διαφερει απο την αποψη οτι το προγραμμα θα τρεξει με της εξης εντολες :

./mngstd -i inputfile -c configfile
./mngstd -c configfile -i inputfile
./mngstd -i inputfile
./mngstd -c configfile
./mngstd

Για compilation γραφεται απλα make εφοσον υπαρχει makefile.
